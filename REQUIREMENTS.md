# Context

This application is designed to handle a gym enterprise. Each enterprise will have an instance of this application. Each enterprise could have some gyms, customers which could join to any enterprise gym and employees.
To inscribe to the enterprise as customer you will have different rates which are global for all gyms. We a customer got inscribed, a contract is created between enterprise, customer and rate, with some additional information about employee and gym where contract got signed.

This application will have a backstage that allow to the manager handles previous aspects of the enterprise.

# Functional Requirements

## Core

- You should see all gyms in the enteprise, update, create and delete them. You could filter gym by some core criteria like name, location and others
- You should list all employees, update, create and delete them. In addition, you could assign to employee the gym where he works. Support core fitlers too
- You should list all rates, update, create and delete them. Also it should support essential filters
- Contract should be accesible on read mode from specific customer. In addition, you could finish a contract. This action will be required before you could delete a customer

## Additional

- Show a ranking of employess which has recruited more customer per month (with month selector)
- Show an analytics page where you have really useful insights like top 3 gyms with more customers, total gyms, total employees, total customers, top 3 most popular rates, avg of new customer per month.
- A real time notification each a new customer has signed a new contract

# Non Functional Requirements

## Core

- Entity deletion should be logic deletion
- SPA as web application
- Independent backend to give support to web application
- Unit Test coverage >80% (frontend and backend)

## Additional
